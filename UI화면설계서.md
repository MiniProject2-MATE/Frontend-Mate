## 3. 주요 화면 상세 설명

### 3.1 메인 페이지 (SCR-001)

#### 3.1.1 화면 개요
| 항목 | 내용 |
| --- | --- |
| **연결 API** | `GET /api/projects?keyword={keyword}&category={PROJECT|STUDY}&sort=latest&page={n}&size=15` |

#### 3.1.3 UI 컴포넌트 매핑
| 화면 요소 | React 컴포넌트 | 라이브러리 컴포넌트 | 비고 |
| --- | --- | --- | --- |
| 검색 바 | **페이지 내 직접 구현** | `Paper`, `TextField`, `Button` (MUI) | 키워드 실시간 검색 |
| 카테고리 필터 | **페이지 내 직접 구현** | `Button` (MUI) | 전체/STUDY/PROJECT |
| 모집글 카드 | `PostCard` | `Card` (MUI) | **공통 컴포넌트** (D-Day 계산 포함) |
| 페이지네이션 | **페이지 내 직접 구현** | `Pagination` (MUI) | 페이지당 15개 기준 이동 |

#### 3.1.4 화면 상태 목록
| 상태 | 표시 내용 | 트리거 |
| --- | --- | --- |
| **로딩 중** | 15개의 Skeleton Card 표시 | `isLoading: true` |
| **데이터 있음** | 모집글 카드 그리드 표시 | `posts.length > 0` |
| **데이터 없음** | "검색 결과가 없습니다." | `posts.length == 0` |

#### 3.1.5 사용자 인터랙션 흐름
- **키워드 입력**: keyword 상태 실시간 변경 및 API 재호출.
- **카테고리 클릭**: 필터링 조건 변경 및 목록 갱신.
- **페이지네이션**: 페이지 이동 후 섹션 상단(`#new-opportunities`)으로 스크롤 이동.

### 3.2 모집글 상세 페이지 (SCR-002)

#### 3.2.1 화면 개요
| 항목 | 내용 |
| --- | --- |
| **주요 기능** | 상세 조회, 팀장/멤버 확인, 지원하기(이동), 삭제/마감(작성자), 지원 취소(지원자) |
| **연결 API** | `GET /api/projects/{id}`, `GET /api/users/me`, `PATCH /api/projects/{id}/close`, `DELETE /api/projects/{id}`, `DELETE /api/applications/{id}` |

#### 3.2.3 UI 컴포넌트 매핑
| 화면 요소 | React 컴포넌트 | 라이브러리 컴포넌트 | 비고 |
| --- | --- | --- | --- |
| 브레드크럼 | `Breadcrumb` | `Breadcrumbs` (MUI) | **공통 컴포넌트** (홈 > 상세) |
| 게시글 헤더 | **페이지 내 직접 구현** | `Typography`, `Chip` (MUI) | 제목 및 모집 상태 중앙 배치 |
| 상세 모집 정보 | **페이지 내 직접 구현** | `Stack`, `Box` (MUI) | 인원, 방식, 마감일 카드 섹션 |
| 팀장 정보 카드 | **페이지 내 직접 구현** | `Avatar`, `Chip` (MUI) | 오너 닉네임 및 포지션 표시 |
| 참여 현황바 | **페이지 내 직접 구현** | `Box` (MUI) | 참여 인원 비례 프로그레스 바 |
| 상호작용 버튼 | `CustomButton` | `Button` (MUI) | **공통 컴포넌트** (지원/삭제/수정) |
| 삭제 확인 모달 | `ConfirmModal` | `Dialog` (MUI) | **공통 컴포넌트** (확인 모달) |

#### 3.2.5 지원 및 관리 버튼 상태 (동적 렌더링)
| 조건 | 버튼 문구 | 버튼 상태 | 비고 |
| --- | --- | --- | --- |
| 미인증 | **지금 지원하기** | 활성 | 로그인 리다이렉트 |
| 작성자 | **작성하신 공고입니다** | 비활성 | 하단에 [수정], [삭제] 노출 |
| 모집마감 | **모집 종료** | 비활성 | `status != 'RECRUITING'` |
| 이미지원 | **이미 지원한 글입니다!!** | 비활성 | `hasApplied == true` |

#### 3.2.6 사용자 인터랙션 흐름

[지원하기 버튼 클릭]
    → 미인증 시: "로그인이 필요한 서비스입니다." 토스트 알림 표시 후 로그인 페이지(`/login`)로 이동한다. (성공 후 복귀를 위한 state 포함)
    → 작성자 본인일 시: "본인이 작성한 공고에는 지원할 수 없습니다." 토스트 알림을 표시한다.
    → 정상 시: 모집글 지원 페이지(`/posts/:id/apply`)로 이동한다.

[수정 버튼 클릭] (작성자 전용)
    → 해당 게시글의 정보를 수정할 수 있는 모집글 관리 페이지(`/posts/:id/edit`)로 이동한다.

[삭제 버튼 클릭] (작성자 전용)
    → "정말로 이 게시글을 삭제하시겠습니까? 삭제 후에는 복구할 수 없습니다." 내용의 `ConfirmModal`을 띄운다.
    → 확인 클릭 시: `DELETE /api/projects/:id` API를 호출한다.
    → 성공: "게시글이 삭제되었습니다." 토스트 알림 표시 후 메인 페이지(`/`)로 이동한다.

[상태 확인 및 버튼 표시]
    → 이미 지원한 글인 경우: 버튼 텍스트가 "이미 지원한 글입니다!!"로 변경되며 버튼이 비활성화된다.
    → 모집 마감인 경우: 버튼 텍스트가 "모집 종료"로 변경되며 버튼이 비활성화된다.

### 3.3 모집글 관리 페이지 (SCR-003)

#### 3.3.1 화면 개요
| 항목 | 내용 |
| --- | --- |
| **화면명** | 모집글 관리(수정) 페이지 |
| **URL** | `/posts/:id/edit` |
| **화면 타입** | 입력/수정 |
| **권한 레벨** | 로그인 필요 (글 작성자/OWNER 전용) |
| **주요 기능** | 기존 모집 정보 수정, 모집 조기 마감, 게시글 삭제, 수정 체크리스트 확인 |
| **연결 API** | `GET /api/projects/{id}`, `PUT /api/projects/{id}`, `DELETE /api/projects/{id}`, `PATCH /api/projects/{id}/close` |

#### 3.3.3 UI 컴포넌트 매핑
| 화면 요소 | React 컴포넌트 | 라이브러리 컴포넌트 | 비고 |
| --- | --- | --- | --- |
| 브레드크럼 | `Breadcrumb` | `Breadcrumbs` (MUI) | **공통 컴포넌트** (홈 > 탐색 > 수정) |
| 기본 정보 폼 | **페이지 내 직접 구현** | `Paper`, `TextField` (MUI) | 모집 유형 선택(PROJECT/STUDY) 및 제목 수정 |
| 모집 조건 설정 | **페이지 내 직접 구현** | `Select`, `Autocomplete` (MUI) | 인원, 마감일자(년/월/일), 기술 스택 수정 |
| 상세 소개 입력 | **페이지 내 직접 구현** | `TextField` (MUI) | 멀티라인 텍스트 필드를 이용한 내용 수정 |
| 상호작용 버튼 | `CustomButton` | `Button` (MUI) | **공통 컴포넌트** (수정 완료/취소 등) |
| 게시글 삭제 버튼 | `CustomButton` | `Button` (MUI) | **공통 컴포넌트** (위험 구역 전용 버튼) |
| 삭제 확인 모달 | `ConfirmModal` | `Dialog` (MUI) | **공통 컴포넌트** (확인 모달) |
| 수정 체크리스트 | **페이지 내 직접 구현** | `Card`, `CheckCircle` (MUI) | 입력 유효성에 따른 실시간 체크 상태 표시 |

#### 3.3.6 사용자 인터랙션 흐름

[조기 마감 버튼 클릭]
    → "정말 이 모집글을 조기 마감하시겠습니까?" 확인 모달 노출
    → '마감하기' 클릭 시: `PATCH /api/projects/:id/close` API 호출
    → 성공: "모집이 마감되었습니다." 토스트 표시 후 상세 페이지 이동

[삭제 버튼 클릭]
    → "정말 이 게시글을 삭제하시겠습니까? 삭제된 게시글은 복구할 수 없습니다." 확인 모달 노출
    → '삭제하기' 클릭 시: `DELETE /api/projects/:id` API 호출
    → 성공: "삭제되었습니다." 토스트 표시 후 메인 페이지(`/`) 이동

[수정 완료 버튼 클릭]
    → 필수 항목(*) 누락 시: "모든 필수 항목(*)을 입력해주세요." 경고 토스트 출력
    → 정상 시: `PUT /api/projects/:id` API 호출
    → 성공: "수정이 완료되었습니다! 🚀" 토스트 표시 후 상세 페이지 이동

### 3.8 마이페이지 상세 (SCR-008)
