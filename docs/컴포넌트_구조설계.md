# 컴포넌트 구조설계

# Mate

> 이 문서는 `컴포넌트 라이브러리`에서 정의된 **물리적 파일 구조**와 **담당자 배분**을 기반으로 MATE 프로젝트의 컴포넌트 조립 방식과 데이터 흐름을 정의합니다.
> 

---

## 1. 컴포넌트 분류 전략

`컴포넌트 라이브러리`의 합의 사항에 따라 다음과 같이 이원화하여 관리합니다.

- **Layout 컴포넌트**: 모든 페이지에서 고정적으로 쓰이는 전체 틀 (`Header`, `Footer`)
- **Common 컴포넌트**: 여러 페이지에서 반복적으로 사용되는 UI 부품 (`Button`, `Tag`, `Avatar`, `CommentItem` 등)
- **Page 컴포넌트**: 각 URL 경로에 매핑되는 독립적인 화면 (`MainPage`, `PostDetailPage` 등)

---

## 2. 디렉토리 구조 (Directory Tree)

`컴포넌트 라이브러리` 섹션 3의 파일 구조를 100% 따릅니다.

```
src/
├── api/                # Axios instance 및 도메인별 API 함수
├── components/         # 재사용 가능한 UI 부품
│   ├── layout/         # Header.jsx, Footer.jsx
│   └── common/         # Button.jsx, Badge.jsx, Tag.jsx, Avatar.jsx,
│                       # FormInput.jsx, Pagination.jsx, Breadcrumb.jsx,
│                       # CommentItem.jsx, ToastMessage.jsx, SkeletonCard.jsx
├── pages/              # 라우팅되는 페이지 컴포넌트 (Container 역할)
│   ├── MainPage.jsx    # 메인 모집글 탐색
│   ├── PostDetailPage.jsx # 모집글 상세 정보
│   ├── LoginPage.jsx   # 로그인
│   ├── RegisterPage.jsx # 회원가입
│   ├── PostWritePage.jsx # 모집글 작성
│   ├── PostEditPage.jsx # 모집글 수정
│   ├── MyPage.jsx      # 마이페이지 활동 내역
│   └── BoardPage.jsx   # 팀 전용 게시판 (추후 구현)
├── store/              # 🐻 Zustand 기반 전역 상태 관리
│   ├── authStore.js    # 로그인/사용자 인증 정보
│   └── postStore.js    # 모집글 목록/상세 데이터
├── styles/             # 전역 스타일 및 MUI Theme
└── utils/              # 공통 유틸리티 (날짜 포맷팅 등)
```

## 3. 컴포넌트 계층 구조 (Hierarchy)

### 3.1 레이아웃 조립 구조

```
App.jsx (Router)
└── MainPage (Page)
    ├── Header (Layout)
    │   ├── Button (Common) - [로그인/Get Started]
    │   └── Avatar (Common) - [로그인 후 프로필]
    ├── (각 페이지별 Main Content 영역)
    │   ├── SearchBar (Page Inner)
    │   ├── Tag (Common) - [기술 스택 필터]
    │   ├── SkeletonCard (Common) - [로딩 시]
    │   ├── PostCard (Page Inner)
    │   └── Pagination (Common)
    ├── Footer (Layout)
    └── ToastMessage (Common) - [상태 알림]
```

## 4. 데이터 및 상태 관리 (State Flow)

### 4.1 데이터 흐름 원칙

1. **Page 중심**: `pages/` 폴더의 컴포넌트가 API를 호출하고 데이터 상태를 관리합니다.
2. **Props 전달**: `common/`의 부품들은 Page로부터 데이터를 Props로 전달받아 그리기만 하는 '순수 컴포넌트'로 유지합니다.
3. **Zustand 활용**: 로그인 상태(`authStore`)와 같이 여러 페이지에서 공유가 필요한 데이터만 전역 스토어에 보관합니다.

| 스토어 | 관리 데이터 | 주요 사용처 |
| --- | --- | --- |
| **authStore** | `token`, `user` | Header(로그인 여부), MyPage, PostWrite |
| **postStore** | `posts`, `currentPost` | MainPage(목록), PostDetailPage(상세) |

## 5. 라우팅 및 권한 맵 (Router Map)

`UI 화면 설계서` 및 `REST API 설계서`의 권한 체계를 반영합니다.

| URL Path | Component | 필요 권한 | 비고 |
| --- | --- | --- | --- |
| `/` | `MainPage` | GUEST | 모집글 검색 및 필터링 |
| `/login` | `LoginPage` | GUEST | - |
| `/register` | `RegisterPage` | GUEST | 초기 기술 스택 설정 포함 |
| `/projects/:id` | `PostDetailPage` | GUEST | 상세 정보 조회 및 지원 |
| `/projects/:id/board` | `BoardPage` | **MEMBER/OWNER** | 팀 전용 소통 공간 |
| `/projects/new` | `PostWritePage` | USER | - |
| `/projects/:id/edit` | `PostEditPage` | **OWNER** | 방장 전용 수정 페이지 |
| `/mypage` | `MyPage` | USER | 활동 이력 통합 조회 |
| *(외부 경로)* | *Admin Page* | **ADMIN** | **Thymeleaf 별도 운영 (React 제외)** |

---

## 6. 개발 가이드라인

1. **파일 확장자**: `컴포넌트 라이브러리` 정의에 따라 `.jsx`를 사용합니다.
2. **컴포넌트 정의**: `export default function Name() { ... }` 형식을 유지합니다.
3. **스타일 적용**: `디자인 토큰`에서 정의된 색상 상수를 CSS 변수나 MUI Theme에 등록하여 사용합니다.
4. **로딩 처리**: 데이터 로딩 시에는 `common/SkeletonCard.jsx`를 활용하여 사용자 경험을 높입니다.
